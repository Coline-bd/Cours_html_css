Les Media Queries CSS

Le Responsive Design consiste à créer des sites web qui s'adaptent automatiquement à la taille de l'écran de l'utilisateur. Les Media Queries sont l'outil principal pour y parvenir.

1. La balise Viewport

Pour que les Media Queries fonctionnent sur mobile, cette balise est obligatoire dans le <head> :

<meta name="viewport" content="width=device-width, initial-scale=1.0">

2. Syntaxe de base

Une media query définit une règle qui ne s'applique que si une condition est remplie (généralement la largeur de l'écran).

/* Style par défaut (Mobile First ou Desktop First) */
.card { width: 100%; }

/* Si l'écran fait au moins 768px de large */
@media (min-width: 768px) {
    .card { width: 50%; }
}

/* Si l'écran fait au moins 1024px de large */
@media (min-width: 1024px) {
    .card { width: 25%; }
}

3. Mobile First vs Desktop First

  • Desktop First : On code pour les grands écrans, puis on réduit avec max-width.
  • Mobile First (Recommandé) : On code pour mobile, puis on ajoute des règles pour les grands écrans avec min-width.

4. Points de rupture (Breakpoints) courants

Il n'y a pas de règle absolue, mais voici les standards souvent utilisés :

  • 480px : Mobiles en portrait
  • 768px : Tablettes
  • 1024px : Ordinateurs portables / petits écrans
  • 1200px : Écrans larges (Desktop)

5. Autres types de Media Queries

On peut aussi cibler d'autres critères que la largeur :

  • orientation: landscape (paysage) ou portrait
  • print : styles spécifiques pour l'impression du document
  • prefers-color-scheme: dark : pour le mode sombre de l'utilisateur
@media print {
    nav, footer { display: none; } /* On cache le menu à l'impression */
}

@media (prefers-color-scheme: dark) {
    body { background-color: #333; color: white; }
}
